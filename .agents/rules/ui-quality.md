# TripLens UI Quality

TripLens should feel like a polished travel insight product, not a generic
admin dashboard.

Use these interface principles:

- Mobile-first layout that also uses desktop space well.
- Clear visual hierarchy between overview, selected trip, comparison, charts,
  form, and trip ledger.
- Light spacious surfaces with soft shadows and restrained accents.
- Roughly 8px radii for cards, buttons, inputs, and panels.
- No nested card clutter unless the inner surface is a real repeated item.
- The selected trip and comparison trip choices should be visually obvious.
- Forms should have readable labels, aligned fields, useful grouping, clear
  primary/secondary actions, and comfortable mobile spacing.
- Charts should have nearby text/list summaries so insight is not color-only.
- Buttons and controls should have accessible names.

Prefer improving actual interaction clarity over decorative polish.
